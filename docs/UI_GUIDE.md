# UI 디자인 가이드 — Cloak Alarm

## 디자인 원칙
1. **아침에 쓰는 앱이다.** 눈이 덜 떴을 때도 한 번에 읽힌다. 작은 텍스트, 복잡한 레이아웃 금지.
2. **알람 시간이 전부다.** 메인 화면의 80%는 시간 표시. 나머지는 최소화.
3. **날씨 상태가 한눈에 들어와야 한다.** 아이콘 + 색상으로 눈/비/맑음을 즉시 인식.
4. **인터랙션은 단순하게.** 탭, 슬라이드 — 복잡한 제스처 없음. 반수면 상태 사용자 기준.
5. **모든 상태에 UI가 있다.** 로딩, 에러, 빈 상태, 권한 없음 — 각각 명확한 화면.

## AI 슬롭 안티패턴 — 절대 하지 마라
| 금지 사항 | 이유 |
|-----------|------|
| `backdropFilter: blur` | glass morphism은 AI 템플릿의 가장 흔한 징후 |
| gradient text | AI SaaS 랜딩의 1번 특징 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 글로우 애니메이션 (shadow glow) | 네온 글로우 = AI 슬롭 |
| 배경 blur 원형 orb | 모든 AI 랜딩에 있는 장식 |
| 모든 카드에 동일한 큰 radius | 균일한 둥근 모서리는 템플릿 느낌 |
| 로딩 스피너에 불필요한 색상 | 흰색 스피너면 충분 |

## 화면별 상세 레이아웃

### 1. 온보딩 — 알림 권한

```
┌─────────────────────────────┐
│                             │
│         🔔                  │  ← 아이콘, fontSize 64
│                             │
│   알람이 울리려면           │
│   알림 권한이 필요해요      │  ← fontSize 22, bold
│                             │
│   권한이 없으면 알람이      │
│   작동하지 않습니다         │  ← fontSize 14, #a3a3a3
│                             │
│  [허용하기 (권장)]          │  ← Primary 버튼
│  [나중에]                   │  ← Text 버튼, #525252
│                             │
└─────────────────────────────┘
```
"나중에" 탭 시: Step 2(위치 권한)로 진행. 메인 화면 진입 시:
- 상단 고정 배너 노출: 빨간 배경, "🔔 알림 권한이 없어요. [설정 열기]", 닫기 불가, 매 세션 표시
- 알람 시간/토글/저장 버튼 전체 비활성(opacity 0.5)
- 탭 시 "알림 권한을 먼저 허용해주세요" 툴팁
- 설정 앱에서 권한 허용 후 앱 복귀 시: 권한 상태 자동 갱신 → 배너 fade-out + 컨트롤 활성화 + "권한이 활성화됐어요!" 토스트

```
│                             │
│  ──────────────────────     │
│                             │
│  [월] [화] [수] [목] [금]   │  ← 요일 토글
│  [토]                       │
│                             │
│  ──────────────────────     │
│                             │
│  기준 알람        06:40    │  ← 탭하면 TimePicker
│  눈/비 시 일찍    20분 ▸   │  ← 탭하면 분 선택
│                             │

│                             │
└─────────────────────────────┘
```

**날씨 없음 상태** (API 실패):
```
│  ⚠️ 날씨 정보 없음  [재시도]│  ← 상단 배너
│         06:40               │  ← 기본 시간 (포인트 컬러 없음)
│     날씨 확인 실패          │  ← 서브텍스트
```

**알람 비활성 상태**:
```
│         06:40               │  ← 시간, #525252 (흐리게)
│         비활성              │  ← 서브텍스트
```

**로딩 상태** (날씨 체크 중):
```
│         06:40               │
│     날씨 확인 중 ...        │  ← 서브텍스트, ActivityIndicator 인라인
```
- 2초 미만: 인라인 스피너만 표시
- 8초 타임아웃 후 3회 재시도 실패: 인라인 스피너 사라짐 + 에러 배너 노출
- 로딩 중 사용자가 TimePicker 조작 가능 → 기존 로딩 취소 후 새 설정으로 재요청

**첫 알람 설정 완료 피드백**:
```
│    ✓ 06:20 알람이 설정되었어요  │  ← 2초 Toast (하단)
│         06:20               │  ← 날씨 배지 fade-in 애니메이션
│     눈 예보 · 20분 일찍 ⚡  │
```

```

  height: 60
  backgroundColor: '#1a1a1a'
  borderRadius: 30
  overflow: 'hidden'

  width: 52, height: 52
  borderRadius: 26
  backgroundColor: '#ffffff'
  // spring animation: tension 100, friction 8

track text: '밀어서 해제'
  fontSize: 14, color: '#525252'
  포지션: 중앙 (thumb가 위로 오면 가려짐)

완료 임계값: 컨테이너 너비의 80% 이상
완료: Heavy haptic + 화면 전환
미완료(<80%) 해제: spring 애니메이션으로 원위치 스냅백, haptic 없음
세로 방향 스와이프(Y축 > X축): 무시
500ms 이상 완료 위치 홀드: 해제 없이도 자동 완료 처리 (Heavy haptic)
접근성: accessibilityAction "두 손가락 더블탭으로 알람 해제" 추가
```

## 빈 상태 (Empty State)

메인 화면에서 알람이 한 번도 설정되지 않은 경우:
```
┌─────────────────────────────┐
│                             │
│         ⏰                  │  ← fontSize 64
│                             │
│      알람을 설정해보세요    │  ← fontSize 20
│  아래에서 시간을 탭해주세요 │  ← fontSize 14, #a3a3a3
│                             │
│      [+ 알람 설정하기]      │  ← Primary 버튼. 탭 시 TimePicker 포커스
│                             │
└─────────────────────────────┘
```

첫 알람 설정 완료 시: "✓ [HH:MM] 알람이 설정되었어요" 2초 Toast (하단) + 날씨 배지 fade-in.
모든 요일 비활성 시: 시간 영역에 "알람 없음" (#525252) + "활성화된 요일이 없어요" 서브텍스트 표시.

| SlideToDisarm VoiceOver 대체 | `accessibilityActions: [{ name: 'activate', label: '알람 해제' }]` 추가. 두 손가락 더블탭으로 해제 가능. |
| 요일 토글 | `accessibilityLabel="월요일, 활성"` / `"월요일, 비활성"` (상태 포함) |
| Dynamic Type | `allowFontScaling={false}` — 알람 시간은 크기 고정 (레이아웃 깨짐 방지) |
| 최소 터치 영역 | 모든 버튼 `minHeight: 44, minWidth: 44` (iOS HIG 기준) |
| 색상만으로 정보 전달 금지 | 날씨 상태는 아이콘 + 색상 + 텍스트 병행 |
| 권한 배너 | `accessibilityRole="alert"`, `accessibilityLiveRegion="assertive"` |
| 설정 V2 비활성 행 | `accessibilityState={{ disabled: true }}` + `accessibilityHint="V2에서 지원될 예정입니다"` |

## 화면 방향

- **Portrait only** 고정. 가로 모드 미지원.
