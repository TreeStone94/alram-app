# PRD: 날씨 연동 스마트 알람 앱 (Cloak Alarm)

## 목표
날씨에 맞게 알람 시간을 자동 조정하고, 기상 후 오늘 날씨에 맞는 옷차림을 TTS로 안내해주는 스마트 알람 앱.
"알람 + 날씨 + 옷차림 추천"을 하나의 아침 루틴으로 통합.

---

## 사용자

| 단계 | 대상 | 목표 |
|------|------|------|
| 1차 | 개발자 본인 (iOS, 한국 거주) | 매일 실사용 → 신뢰도 검증 |
| 2차 | 지인 10명 베타 | 일주일 이상 사용 → UX 피드백 |
| 3차 | App Store 한국 사용자 | 아침 루틴에 신경 쓰는 20-40대 |

## 핵심 기능 (MVP V1)

### 1. 날씨 연동 스마트 알람

#### 정상 동작
- 사용자가 기준 알람 시간 설정 (예: 6:40)
- 눈/비 예보 시 미리 일어나는 시간 별도 설정 (기본값: 20분)
- 알람 설정 저장 시점에 기상청 API로 강수 예보 확인
- 강수형태(PTY) 코드: 1=비, 2=비/눈, 3=눈, 4=소나기 → 이 중 하나면 일찍 알람
- 결과:
  - 강수 예보 있음 → 기준 시간 - 미리 일어나기 분 = 최종 알람 시각 (예: 6:20)
  - 강수 예보 없음 → 기준 시간 그대로 (예: 6:40)
- 요일별 활성/비활성 토글 (기본: 월~금 활성)

#### 엣지케이스 & 에러 처리

| 케이스 | 처리 방식 |
|--------|----------|
| 날씨 API 호출 실패 (네트워크 없음) | 캐시 데이터 사용 (24시간 이내). 캐시도 없으면 기본 알람 시간으로 설정. 화면에 "날씨 확인 실패 — 기본 시간으로 설정됨" 배너 표시 |
| 날씜 API 호출 실패 (서버 오류 5xx) | 최대 3회 재시도 (exponential backoff: 1s, 2s, 4s). 모두 실패 시 캐시 or 기본 시간 |
| GPS 위치 권한 거부 | 수동 주소 입력 화면으로 이동. 한국 시/군/구 선택 UI. 선택한 주소로 격자 좌표 계산 |
| GPS 위치 획득 타임아웃 (10초 초과) | 마지막 저장된 위치 사용. 없으면 수동 주소 입력 |
| 기상청 API 응답 파싱 오류 (예상치 못한 필드) | 파싱 실패 → 기본 알람 시간. 오류 로그 로컬 저장 |
| 알람 시각이 현재 시각보다 과거 (예: 자정 직후 설정) | 다음 날 같은 시각으로 스케줄. 사용자에게 "내일 [시각] 알람" 확인 메시지 |
| 미리 일어나기 분 적용 시 알람이 자정 이전으로 당겨지는 경우 (예: 00:10 → -10분) | 사용자에게 확인 다이얼로그 표시. "[기본 시간으로 설정] vs [전날 23:50으로 설정]" 명시적 선택 필요. 자동 진행 없음. 메인 화면에서 해당 시간은 ⚠️ 아이콘 + 다른 색상으로 강조. |
| Notification 스케줄 실패 (OS 권한 없음) | 알림 권한 요청 화면으로 이동 |
| Notification 스케줄 실패 (알 수 없는 오류) | 사용자에게 Toast: "알람 설정에 실패했어요. 다시 시도해주세요" |
| 폰 재부팅 후 알람 스케줄 소실 | 앱 재실행 시 저장된 AlarmConfig로 재스케줄. iOS는 앱 실행 시에만 복원 가능 — 온보딩에서 명시. 복원 성공 시 5초 배너 "✓ 알람 복구됨. [HH:MM]에 울어요" 표시. 이미 지난 알람이면 다음 활성 요일로 재계산. |
| 앱 업데이트 후 알람 스케줄 소실 | 앱 실행 시 AsyncStorage AlarmConfig 확인 → 재스케줄 시도 |
| BackgroundFetch 미실행 (iOS OS가 연기) | 알람 설정 시점 스케줄이 그대로 유효. BackgroundFetch는 보조 수단 |
| BackgroundFetch 실행 시 API 실패 | 기존 스케줄 유지. 변경하지 않음 |
| 알람 울리는 시간에 앱이 foreground | alarm-ring 화면으로 즉시 전환 |
| 알람 울리는 시간에 앱이 background | Notification 발화 → 사용자 탭 → alarm-ring 화면 |
| 알람 울리는 시간에 앱이 완전 종료 | Notification 발화 → 사용자 탭 → 앱 cold start → alarm-ring 화면 |
| 알람 울리는 시간에 폰이 무음 모드 | 소리 없이 진동만. 온보딩에서 "알람 앱 특성상 무음 해제 권장" 명시 |
| 알람 울리는 시간에 DND(방해금지) 모드 | iOS는 일반 앱이 DND/Focus 예외를 프로그래밍으로 등록 불가. 사용자가 직접 설정해야 함. 앱은 설정 앱 딥링크([UIApplication openSettingsURLString]) 버튼 제공. 온보딩 무음 안내 화면에서 명시: "방해금지 모드를 사용하신다면 [설정 > 집중 모드 > 허용된 앱]에서 이 앱을 추가해주세요." |

### 2. 기상 후 TTS 날씨 브리핑

#### 정상 동작
- 알람 Notification 탭 → 앱 foreground 진입 → alarm-ring.tsx 표시
- 화면 진입 즉시 TTS 자동 재생 시작 (딜레이 500ms 후 — 화면 렌더링 완료 대기)
- TTS 내용 = [스타일 prefix] + [온도 멘트] + [강수 멘트(있을 때)] + [일교차 경고(있을 때)]
- 재생 완료 후 화면에 날씨 카드 표시 유지
- 슬라이드로 알람 해제

#### TTS 멘트 생성 규칙 (온디바이스 템플릿)

**온도별 기본 멘트** (현재 기온 TMP 기준):
| 온도 | 멘트 |
|------|------|
| 28°C 이상 | "반팔 한 장이면 충분해요" |
| 20-27°C | "얇은 긴팔이 딱 좋아요" |
| 12-19°C | "아침엔 선선하니 가디건 챙기세요" |
| 4-11°C | "꽤 쌀쌀해요, 코트나 패딩 입으세요" |
| 4°C 미만 | "오늘 진짜 추워요, 두꺼운 패딩 필수" |

**일교차 보정** (TMX - TMN >= 10°C **AND** TMX >= 15°C):
→ "낮엔 덥지만 저녁엔 쌀쌀해요, 겉옷 챙기세요"
- TMX < 15°C 조건 추가 이유: 영하권(최고 4°C/최저 -6°C)에서 "낮엔 덥지만" 멘트가 부적절함

**강수 보정** (PTY != 0):
| PTY | 멘트 추가 |
|-----|---------|
| 1 (비) | "오늘 비 와요, 우산 꼭 챙기세요" |
| 2 (비/눈) | "비나 눈이 섞여 내려요, 우산이랑 미끄럼 조심" |
| 3 (눈) | "눈 와요! 미끄러우니까 조심하고 두껍게 입으세요" |
| 4 (소나기) | "소나기 예보 있어요, 접이식 우산 챙기세요" |

**V1 스타일 prefix** (친구형):
→ "야~ 일어나! [기본멘트]. [강수멘트]. [일교차멘트]"

**완성 예시**:
→ "야~ 일어나! 오늘 진짜 추워요, 두꺼운 패딩 필수. 눈 와요! 미끄러우니까 조심하고 두껍게 입으세요. 낮엔 덥지만 저녁엔 쌀쌀해요, 겉옷 챙기세요."

#### TTS 엣지케이스 & 에러 처리

| 케이스 | 처리 방식 |
|--------|----------|
| 알람 해제 시점에 날씨 캐시 없음 (API 실패 + 캐시 만료) | "날씨 정보를 가져오지 못했어요. 창문 열어서 직접 확인해보세요!" (fallback 멘트) |
| AVSpeechSynthesizer 오류 | 에러 무시, 화면 날씨 카드만 표시 (TTS 없이 진행). 날씨 스크립트 텍스트를 화면에 자막으로 표시. |
| TTS 재생 60초 초과 (hang 상태) | 화면에 [건너뛰기] 버튼 노출. 탭 시 TTS 중단 + 날씨 카드 + SlideToDisarm 유지. |
| 기온 데이터 null 또는 비정상 값 | 온도 멘트 생략, 강수/일교차 멘트만 사용 |
| 폰 볼륨 0 | TTS 재생 시도. 결과는 사용자 책임 (볼륨 컨트롤 없음) |
| 사용자가 TTS 재생 중 알람 해제 슬라이드 | TTS 즉시 중지 후 해제 처리 |
| 앱 foreground 진입 후 TTS 재생 전 백그라운드로 나감 | TTS 재생 중단. 다시 foreground 시 재생 시도하지 않음 (이미 해제 의도로 간주) |

### 3. 권한 플로우 (온보딩)

앱 최초 실행 시 순서대로 요청:

```
Step 1: 알림 권한 요청
  → [허용하기]: iOS 시스템 권한 다이얼로그 표시 → 허용 시 Step 2 진행
  → [나중에]: Step 2로 진행 (알람 기능 비활성 상태).
             메인 화면 진입 시 상단에 고정 배너 표시:
             "🔔 알림 권한이 없어요. [설정 열기]"
             배너는 닫기 불가. 매 앱 실행 시 유지. 권한 허용 후 자동 소멸.
             알람 컨트롤 전체 비활성(opacity 0.5) + 탭 시 툴팁 "알림 권한을 먼저 허용해주세요"
             재요청 기회: 주 1회 또는 알람 설정 시도 시 전체화면 재안내.
  → 권한 거부(iOS 시스템에서) 시: 위와 동일 (배너 + 비활성).
             설정 복귀 후 앱 포그라운드 진입 시 권한 상태 자동 갱신.
             허용으로 변경됐으면: 배너 fade-out + 알람 컨트롤 활성화 + "권한이 활성화됐어요!" 토스트.

Step 2: 위치 권한 요청 ("앱 사용 중" 권한)
  → [위치 허용하기]: iOS 시스템 권한 다이얼로그 → 허용 시 Step 3 진행
  → [직접 주소 입력하기]: 수동 주소 picker 화면으로 이동.
     수동 주소 picker: 시/도 선택 → 시/군/구 선택 (2단계 드롭다운).
     선택 완료 → "날씨 정보를 [주소]로 설정했어요" 확인 메시지 → Step 3 진행.
     선택한 주소는 AsyncStorage에 저장. 설정 화면에서 "현재: [주소]"로 표시 및 변경 가능.
  → 나중에 선택지 없음 — 반드시 둘 중 하나 선택 후 진행.
  → 추후 설정에서 GPS 권한 허용 시 자동으로 GPS 모드로 전환.
  → 위치 변경(여행 등) 시: 설정 > 위치 > [위치 자동 감지] 또는 [수동 변경] 탭.
     변경 후 API 재호출 → 알람 재스케줄 → "위치가 변경됐어요. [도시명] 날씨로 다시 설정했습니다" 확인 메시지.

Step 3: 무음 해제 안내 (권한 아님, 안내 화면)
  → "알람 앱이라 소리가 필요해요. 무음 모드 해제를 권장해요"
     방해금지 모드 사용자를 위한 추가 안내:
     "방해금지 모드를 자주 사용하신다면, 이 앱을 방해금지 예외로 등록해주세요."
     [설정 열기] 링크 제공 (선택 사항).
     [알겠어요, 시작하기] 버튼 후 메인 화면.
```

### 4. 로컬 알림 페이로드 명세

알림 발화 시 사용자에게 보이는 내용:

| 필드 | 값 | 예시 |
|------|-----|------|
| title | "알람이 울었어요" | — |
| body | "[HH:MM] • [날씨 아이콘] [조건 텍스트]" | "06:20 • ❄️ 눈 예보" |
| sound | 시스템 기본 알람음 | — |
| badge | 1 | — |

잠금화면 동작:
- 알림 탭 → 앱 열림 → alarm-ring.tsx 진입 (foreground/cold-start 모두 동일)
- 알림 스와이프 해제 → 알림만 제거. 알람 재스케줄에는 영향 없음. TTS 미재생.
- 스누즈 버튼: **V1에서 명시적으로 제거**. notification categoryIdentifier를 커스텀으로 등록하면
  iOS 기본 스누즈 버튼이 나타나지 않음. 스누즈를 허용하면 AlarmState 머신이 ringing→dismissed
  전환을 놓치고 상태가 깨진 채로 유지되므로 V1 금지.
  구현: `Notifications.setNotificationCategoryAsync('ALARM', [])` — 액션 없이 등록하면 스누즈 제거.

### 5. 다음 알람 표시 (해제 후)

알람 해제 완료 시 다음 활성 요일 알람을 즉시 재스케줄하고 메인 화면에 반영:

- 오늘 이후 가장 가까운 활성 요일 계산 → 해당 날짜 날씨 API 조회 → Notification 재스케줄
- 메인 화면: "다음 [요일] [HH:MM]" fade-in 애니메이션
- 모든 요일 비활성 상태이면: "알람 없음" + "활성화된 요일이 없어요" 표시

---

### 6. 타임존 / DST 정책

- 알람 시간은 **기기 시스템 시간** 기준. 여행 시 기기 시간이 자동 업데이트되면 알람도 함께 조정됨.
- DST 변경: 기기 자동 시간 설정 ON 상태에서는 알람도 자동 보정됨. 별도 처리 불필요.
- 온보딩 안내 문구: "알람 시간은 기기 시간을 따릅니다. 여행 시에는 기기 시간 자동 업데이트를 켜주세요."

|----------|----------|-----------|

| 캐시 만료 + API 실패 | 배너 "오래된 날씨 정보" | 재시도 버튼 |
| TTS 실패 | 무음으로 처리 (에러 표시 없음 — 알람 해제가 더 중요) | — |

### 8. Critical Alerts 계획

iOS Critical Alerts는 무음 모드/DND에서도 알람이 울리는 특별 권한. 건강·안전 카테고리에 해당하는 앱에만 Apple이 부여.

- **V1**: Critical Alerts 미사용. 온보딩에서 "무음 모드에선 알람 소리가 안 울릴 수 있어요. 무음 해제를 권장해요" 안내.
- **App Store 출시 전**: Apple 개발자 포털에서 Critical Alerts 권한 신청 (건강/안전 카테고리, 알람 앱 근거로 신청).
  - 신청 경로: App Store Connect → 앱 정보 → 특별 사용 권한 → Critical Alerts
  - 승인 시: `UNNotificationCategoryOptions.criticalAlert` 활성화 + 알람 Notification에 `criticalSoundVolume` 설정.
  - 미승인 시: 현행 유지 (온보딩 안내만).
- 참조: ADR-006

## 날씨 API 상세 스펙 (기상청 단기예보)

### 사용 API
- 엔드포인트: `https://apis.data.go.kr/1360000/VilageFcstInfoService_2.0/getVilageFcst`
- 파라미터: `serviceKey`, `numOfRows`, `pageNo`, `dataType=JSON`, `base_date`, `base_time`, `nx`, `ny`
- base_time: 0200, 0500, 0800, 1100, 1400, 1700, 2000, 2300 (매일 8회 발표)
- 전날 밤 체크 시 2300 발표 기준 사용

### 주요 응답 필드
| 카테고리 | 설명 | 값 |
|---------|------|---|
| PTY | 강수형태 | 0=없음, 1=비, 2=비/눈, 3=눈, 4=소나기 |
| SKY | 하늘상태 | 1=맑음, 3=구름많음, 4=흐림 |
| TMP | 1시간 기온 (°C) | 숫자 |
| TMX | 일 최고기온 (°C) | 숫자 |
| TMN | 일 최저기온 (°C) | 숫자 |
| REH | 습도 (%) | 숫자 |
| WSD | 풍속 (m/s) | 숫자 |

### API 제약
- 일일 호출 한도: 10,000회 (개인 키 기준)
- 응답 지연: 평균 1-3초, 최대 10초
- 서비스 점검: 매일 04:00-04:10 (점검 중 호출 → 503)
- 호출 시 04:00-04:10이면 이전 캐시 사용 또는 0200 발표 데이터 사용

### API 타임아웃 & 로딩 UX
- 단건 요청 타임아웃: **8초**
- 2초 미만 완료: 인라인 스피너 + "날씨 확인 중..." (메인 화면 서브텍스트 영역)
- 2초 이상 지연: 동일 인라인 유지 (전체화면 오버레이 없음)
- 8초 타임아웃 도달: 자동 재시도(최대 3회, 1s/2s/4s backoff) 시작
- 3회 모두 실패: 기본 알람 시간으로 폴백 + 배너 "날씨를 확인할 수 없어요. 기본 시간으로 설정됐어요" + [재시도] 버튼
- 로딩 중 사용자가 TimePicker 조작: 로딩 취소 → 새 시간 기준으로 재요청
